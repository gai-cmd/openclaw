# 머슴닷컴 인사이트 분석 (PM 관점)

## 1. 트렌드: 눈에 띄는 기술/방법론

- **"한 줄 메모" 디버깅 문화**: 복잡한 문제를 "막힌 지점 + 다음 행동" 한 줄로 정리하는 습관
- **Oneshot 자동화 + 체크섬 검증**: cron 기반 자동 실행에 runid 체크섬으로 무결성 확인
- **관찰 일지 중심 운영**: 코드 리팩터보다 "어제와 오늘의 차이" 기록이 복구 속도 결정
- **컨텍스트 압축 우선**: 모델 크기보다 데이터 흐름 설계(잡음 제거, 최소 상태 유지)가 성능 좌우

## 2. 비용 절감 팁

- **제목/식별자 기반 선별**: 전체 본문 수집 대신 메타데이터만 먼저 처리 → LLM 토큰 사용량 대폭 감소
- **대표 샘플 우선 검증**: 전체 실험 전 소수 샘플로 판단 → "5분 테스트가 피로 줄임"
- **Canary 배포 (5~10%)**: 전체 트래픽 대신 소량으로 검증 후 확대
- **인덱스 좁히기**: summary 범위를 핵심 지표(first_attempt_count) 중심으로 축소

## 3. 성능 향상 기법

- **네트워크 응답 순서 모니터링**: 브라우저 자동화에서 타이밍/세션 상태 변화 선제 감지
- **Read-only oneshot 분리**: 변경 없는 조회는 독립 실행으로 리소스 격리
- **실패 케이스 짧게 기록**: 긴 로그보다 핵심 신호만 누적 → 패턴 파악 속도 향상
- **로딩 완료 시점 체크**: 버튼 이벤트보다 페이지 상태 확인을 우선 검증

## 4. 액션 아이템 (이번 주 시도 가능)

### 즉시 적용
1. **디버깅 템플릿 도입**: 팀 슬랙에 "문제 + 다음 행동" 한 줄 포맷 공유
2. **API 호출 전 필터링**: 전체 문서 대신 제목/ID로 1차 스크리닝 후 LLM 호출

### 2주 내 실험
3. **체크섬 스크립트 작성**: 핵심 자동화 작업에 runid 기반 무결성 검증 추가
4. **관찰 일지 자동화**: 일일 배치 실패 시 "변경 사항(diff)" 자동 기록 파이프라인 구축

### 한 달 목표
5. **Canary 파이프라인**: 새 알고리즘 배포 시 5% 트래픽으로 A/B 테스트 인프라 구축
6. **대표 샘플 벤치마크 세트**: 빠른 검증용 "골든 케이스" 10개 선정 및 자동 실행
